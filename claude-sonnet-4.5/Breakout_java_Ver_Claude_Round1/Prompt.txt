ส่วนที่ 1: คำสั่งสำหรับสร้างโค้ดเกม Breakout
1. วัตถุประสงค์หลัก
สร้างแอปพลิเคชันเกม Breakout ที่สมบูรณ์ด้วยภาษา Java โดยใช้ Swing framework และยึดตามข้อกำหนดของเวอร์ชันพื้นฐาน
2. ข้อกำหนดและกติกาของเกม
หน้าต่างและขนาด
หน้าต่างเกมต้องมีขนาดคงที่: กว้าง 300 พิกเซล และสูง 400 พิกเซล
ชื่อหน้าต่าง (Title) ต้องเป็น "Breakout"
และให้ Bricks ไม่เกินขอบหน้าต่างเกม
องค์ประกอบในเกม
แพดเดิล (Paddle): แท่งสี่เหลี่ยมแนวนอนที่ผู้เล่นควบคุมบริเวณด้านล่างของจอ
ลูกบอล (Ball): ลูกบอล 1 ลูกที่เคลื่อนที่และเด้งกระทบวัตถุต่างๆ
อิฐ (Bricks): กำแพงอิฐจำนวน 30 ก้อน จัดเรียงเป็น 5 แถว 6 คอลัมน์ที่ส่วนบนของจอ
กลไกการเล่นและการจัดการสถานะ
เริ่มเกมอัตโนมัติ: เมื่อรันโปรแกรมขึ้นมา เกมจะต้องเริ่มทำงาน ทันที โดยมีการวาดองค์ประกอบทั้งหมด และลูกบอลจะเริ่มเคลื่อนที่โดยอัตโนมัติ ไม่ต้องรอการกดปุ่มใดๆ
ฟิสิกส์ของลูกบอล:
ลูกบอลต้องเด้งสะท้อนกับกำแพงด้านบน, ซ้าย, และขวาอย่างสมจริง
ลูกบอลต้องเด้งสะท้อนกับแพดเดิลอย่างสมจริง
เมื่อลูกบอลชนอิฐ, อิฐก้อนนั้นจะถูกทำลาย (หายไป) และลูกบอลจะเด้งออกมา
การควบคุมของผู้เล่น
ผู้เล่นควบคุมการเคลื่อนที่ในแนวนอนของแพดเดิลโดยใช้ เฉพาะปุ่มลูกศรซ้ายและขวาเท่านั้น
แพดเดิลต้องหยุดเมื่อเคลื่อนที่ไปสุดขอบของหน้าต่างเกม และต้องไม่เคลื่อนที่ทะลุออกไป
เงื่อนไขการแพ้-ชนะ
การชนะ: ผู้เล่นจะชนะเมื่อทำลายอิฐได้ครบทั้ง 30 ก้อน เมื่อเงื่อนไขนี้เกิดขึ้น: ข้อความ "Victory" ต้องแสดงขึ้นมา, ลูกบอลต้องหยุดนิ่ง, และเกมจะจบลง ผู้เล่นต้องปิดและรันโปรแกรมใหม่เพื่อเล่นอีกครั้ง
การแพ้: ผู้เล่นจะแพ้หากลูกบอลตกเลยขอบจอด้านล่าง เมื่อเงื่อนไขนี้เกิดขึ้น: ข้อความ "Game Over" ต้องแสดงขึ้นมา, ลูกบอลต้องหยุดนิ่ง, และเกมจะจบลง ผู้เล่นต้องปิดและรันโปรแกรมใหม่เพื่อเล่นอีกครั้ง
3. ฟังก์ชันที่ไม่อยู่ในขอบเขต (ห้ามใส่ฟังก์ชันเหล่านี้)
ห้ามมีฟังก์ชันหยุดเกมชั่วคราว (Pause/Unpause)
ห้ามกำหนดให้มีการกดปุ่มใดๆ (เช่น Spacebar) เพื่อเริ่มเกม
ห้ามมีระบบคะแนน, ระบบชีวิต, ไอเท็มพิเศษ หรือด่านหลายด่าน
ห้ามมีปุ่ม "เล่นอีกครั้ง" (Play Again)
4. ข้อกำหนดทางเทคนิค
จัดโครงสร้างโค้ดโดยแบ่งออกเป็นหลายคลาสตามความเหมาะสมเพื่อให้อ่านง่าย
กรุณาสร้างโค้ด Java ที่สมบูรณ์สำหรับทุกคลาสที่จำเป็น และระบุชื่อไฟล์ที่ต้องสร้างด้วย

ส่วนที่ 2: คำสั่งสำหรับสร้าง JUnit Tests
1. วัตถุประสงค์หลัก
หน้าที่ของคุณคือการเขียนชุดการทดสอบ JUnit (Unit Tests) ที่ครอบคลุมสมบูรณ์สำหรับโค้ดเกม Breakout ที่จะได้รับ
2. ข้อกำหนดด้านคุณภาพที่สำคัญที่สุด
ชุดการทดสอบที่สร้างขึ้น จะต้อง เป็นไปตามเกณฑ์ชี้วัดความครอบคลุม (Coverage Criteria) ดังต่อไปนี้:
Statement Coverage: 100%
Branch Coverage: 100%
คุณต้องเขียน Test Case ที่จำเป็นทั้งหมดเพื่อให้แน่ใจว่าทุกบรรทัดของโค้ดและทุกเงื่อนไขที่เป็นไปได้ทุกไฟล์ (เช่น if/else) จะถูกทำงานระหว่างการทดสอบ และต้องเขียนคอมเม้นบอกว่า Test แต่ละตัวทำอะไรให้ชัดเจนเป็นภาษาไทย และให้มีไฟล์ TestRunner สำหรับเรียกไฟล์ JUnit ทั้งหมดในการรันครั้งเดียว และเขียนสรุป test case ทั้งหมดออกมาลิสเป็นข้อๆ บอก 1.ชื่อไฟล์JUnit  2.ชื่อ test ภายในไฟล์ JUnit 3.รายละเอียดการtest ภายในไฟล์ว่าตัวนี้ test อะไรพร้อม Expected Result เป็นตารางให้อ่ายง่ายเป็นภาษาไทย
3. วิธีการ
นี่คือโค้ด Java ของเกม Breakout ที่สมบูรณ์ หน้าที่ของคุณคือส่งมอบโค้ด JUnit5 Test ที่สมบูรณ์ทั้งหมดเพื่อให้บรรลุเป้าหมาย Statement และ Branch Coverage 100% และให้โค้ดสั้นแต่ไม่กระทบประสิทธิภาพ และแยกการไฟล์ JUnit  เป็นหลายๆไฟล์ ตาม classที่จะเทส
